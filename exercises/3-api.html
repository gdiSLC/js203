Setup

  1. Sign up for a Big Huge Thesaurus account
  2. Get an API key. When you will out the form, you'll have to provide a description (10+ words) of your app and a URL. Use whatever you want! Don't click "Include in public app listing". Click "Get API key" and save it onto your computer somewhere.
  3. If you're using Chrome, install the JSON Formatter extension

Understanding The Big Huge Thesaurus API

In this exercise, we're going to be using the Big Huge Thesaurus API.

View the sample API request for 'love' to get a better understanding of how the API works.

Your JSON response will include parts of speech as keys, with each broken down into the following categories:

  • synonyms ("syn")
  • antonyms ("ant")
  • related terms ("rel")
  • similar_terms ("sim")

Not every word will have every part of speech, or every type of relationship. Keep this in mind when you're writing your code or you will get some weird errors.

Instructions

In this exercise, we're going to be using the Big Huge Thesaurus API to query words and add them onto our page.

You can edit the JavaScript file and the HTML file if you want.

When you work with JavaScript APIs, you have to be running a server for your API call to be successful (this is a security thing). There is an easy(ish) way to run a local server on a Mac (instructions if you were curious), but for this class we are going to use JSBin instead. Even if you know how to run a server on your Mac, please use JSBin for this exercise.

  1. Before you try and edit the files get the API working using your API key on your browser. There are sample URLs on your API key page inside of your account.
  2. Open the JSBin starter file. Click File > Clone to make a copy.
  3. Add a click listener (jQuery documentation for click()) for the form button that will:
    • Prevent the form from submitting (documentation)
    • Get the value of the word in the text box
    • Send an ajax request to the Big Huge Word Labs API (documentation)
    • Process the JSON response using a success callback function
  4. In your callback function, iterate through your JSON using the jQuery $.each (documentation)
  5. Display the word however you want (see bonus section for ideas)

Bonus points

  • Try and avoid editing the HTML file
  • What happens if the text box is empty when the user hits "submit"? Add an alert that tells the user to add a word if the box is empty.
  • Add a header element for each part of speech (noun, verb, etc)
  • Add a subheader for each type of word (synonym, antonym, etc)
  • Make a list for each type of word, and add a list item for each word
  • Edit your code so it is not repetitive
