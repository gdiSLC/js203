Exericse 2: JavaScript Snippet APIs

Setup

When you work with JavaScript APIs, you have to be running a server for your API call to be successful (this is a security thing). There is an easy(ish) way to run a local server on a Mac (instructions if you were curious), but for this class we are going to use JSBin instead. Even if you know how to run a server on your Mac, please use JSBin for this exercise.

Instructions

In this exercise, you'll try out using several JS snippet APIs to bring more functionality into the webpage.

Note: some of these steps require you to have accounts on these services. If you don't already have an account, you can either sign up for an account or skip that step.

  1. View the starter file on JSBin - do not try to edit the starter file
  2. Click File > Clone to create a copy of the file
  3. Go to the Google+ button docs and generate the code for a Google+ button with no annotation and a medium size. Insert that so it shows up next to the header of the webpage.
  4. Go to the Tweet button docs and read about the Tweet button. Find the HTML/SCRIPT tag for adding it, and paste that into your webpage, at the bottom near the counter.
  5. Read the Facebook 'Like' button docs and add a 'Like' button for any page you want.
  6. Modify the share URL or HTML according to the docs to specify hash tags and change it so that it doesn't show the counter.
  7. Bonus: Using JavaScript, generate a Tweet button for each video, and display it when the video is clicked. The text for each tweet button should be the video title.

Make sure that you use your console and developer tools to make debugging easier while working on this. Check for errors, and use console.log() to figure out how far your code makes it, and what the values of your variables are along the way. Using JSBin, you'll need to click the "Console" tap in the top middle. Your developer console might print extra information.
